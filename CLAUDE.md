## What this repo is

An **Arke spec-driven, multi-agent workspace** — not an application codebase. There is no
`package.json`, no solution file, no application source, and **no build, run, or test command**.
Execution is driven externally by the Arke coordinator + OpenCode harness, not by commands run here.

The unit of work is the **specification** (`docs/specifications/`), not the code or the ticket.

The product being specified is **Abby's Table** (below). Nothing of it is built yet — the only
product artefact is a design template.

- **`agents/<name>/config.yaml` does not exist.** The scaffold manifest lists six of them, AGENTS.md
  calls them "the authoritative model + provider declaration", and every `.opencode/agents/*.md`
  prompt instructs you to "edit that image". **That directory was never created.** The harness
  registry confirms it: `registry.updated` reports `serves: []`. Don't send anyone to edit a file
  that isn't there — create the image or fix the reference.
- **`.repos/` does not exist** either, despite `.repos/README.md` being in the manifest.
- **This is not a git repository** (`repo.identity` shows empty `remote`/`default`/`head`). The
  workflow assumes branches and pull requests, and AGENTS.md claims "git history preserves the
  previous version" of a regenerated file. It does not. There is no recovery net — back up
  `AGENTS.md` before triggering a researcher refresh.
- **Active harness warning** (`registry.warning: reviewer-models-identical`): reviewers declare
  non-concrete models (`gateway/reviewer-a`). Review independence is unverifiable until these are
  pinned to provider-qualified ids, e.g. `github-copilot/claude-opus-4.8`. All six agents use the
  same `gateway/<name>` placeholder.
- Harness endpoint is `http://127.0.0.1:4119`; the provider profile in `.arke/config.json` points at
  port `4096`. Both numbers are real — don't "fix" one to match the other.

## Conventions

- **Specs** live in `docs/specifications/`, one file per slug. Anatomy: *Why / What changes /
  Requirements / Design / Tasks* — see `specification.template.md`.
- **Frontmatter:** `spec_id` (`SPEC-YYYY-MM-DD-short-slug`), `title`, `status`, `branch`
  (`feat/<short-slug>`), `owner`, `capabilities`, `created`, `updated`.
- **Requirements** use RFC-2119 ("The system SHALL …") with `capability` / `delta: ADDED (<branch>)`
  annotations and `WHEN/THEN/AND` scenarios, tagged `ADDED`/`MODIFIED` with `FR-xx` ids.
- **Definition of done:** all scenarios pass; typecheck and build green; a reviewer signed off.
  (Note there is currently nothing to typecheck or build.)
- **Never hand-edit:** `AGENTS.md`, `docs/specifications/index.md` (`generated: true`), or any file
  in the scaffold manifest — edits are detected by checksum.
- Kebab-case for agent names and directories. Credentials never live in the repo; `.arke/config.json`
  uses a host-resolved `credentialsRef`.
- No linter, formatter, `tsconfig.json`, or test config exists. The `.ts` plugins are consumed by
  the coordinator runtime and are not compiled here.
